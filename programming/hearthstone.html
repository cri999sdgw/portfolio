Solitair

Exercise

Create a game like Hearthstone, programmed in the language C++!

Game Description

The game is played by 2 player. Each having 7 handcards and 7 play fields. The aim is to attack the enemy with the playcards until he is dead (30 Life Points). You can place your handcards into the field, when you have enough mana. The cards are categorisized in Spell-Cards and Creature-Cards. Spell Cards can not be put on the gamefield, but can be cast. Furthermore each player has a graveyard, where all the killed Creature-Cards are placed. You can also sacrifice a Card, what means, you place the card on the graveyard, and gain 1LP
It's a roundbased system, where the players get for each round a certain amount of mana.
The first one to kill the enemy wins.

Game Setup

The game needs a config file, which has the .json format and is parsed via the rapidjson istreamwrapper. Each Creature-Card needs these following attributes:
- name [Name of the Creature-Card]
- mana_cost [The mana what is needed to place the Card]
- damage_points [How much damage the card does, when attacking]
- life_points [How many Life points the Card has]
- shield [If a card is placed in the field and has the shield attribute, the player cannot be attacked]
Each Spell-Card needs to have the following attribute:
- name [The name of the Spell-Card]

User Input

For displaying the commands: help
For attacking: attack #x #y
For placing a Creature-Card: set #x to #y
For casting a Spell-Card: cast #x
For sacrificing a Card: sacrifice #x
For displaying the gamefield: state
For ending your turn: finish
For quitting the game: quit

Environment

The whole game was developed in VSC (Visual Studio Code). It's a console game, which can be played in every UNIX-Console.

Purpose

The whole purpose of the program was, to pass a subject for my Degree in Computer Science. the subject was named "Object Oriented Programming 1". The main Goal of the Subject was, to teach the fundamentals of the programming language C++.

What did I learn?

I learned the fundamentals of the programming language C++. Inlcuding the usage of classes and objects. We also used derived classes, for our 2 different Card Types.

Error handling

Also a big part of the project was, to handle wrong inputs and not allowing to make not valid moves. Speficic error messages were included, to inform the player that i made a mistake. Since the university had a lot of testcases (In specific 98 for error handling), I'm sure the game is bulletproof to almost every user input.
